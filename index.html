Outil de requête SAP automatique

Ce site est une documentation pour l'utilisation de l'Outil de requête SAP automatique fait par Hani Massinissa sous la guidance de Ménard, Francis [2].

Cet outil a pour but de faciliter la recherche et l’annotation des différentes informations liées aux postes responsables, ainsi que la mise à jour des informations des données par l'entremise de SAP. L'outil a été adapté pour les APRTs de MTL, ainsi que leurs postes responsables et leur interface SAP. En ce qui concerne ce site, il a pour but de montrer les fonctions principales de l'outil, ainsi que la procédure pour adapter ce dernier.

Mise à jour du document

La mise à jour du document permet de s'assurer que les informations dans l’Excel sont à jour avec SAP

Ouvrir SAP

Pour que le code puisse fonctionner, il faut préalablement ouvrir l’interface SAP. Une fois SAP Logon ouvert, choisir dans les accès rapides (les accès rapides ne nécessitent pas de mot de passe) l’élément « Production » (il est possible de le chercher avec le filtre « P01 »)

Lancer le script VBA

Le script peut prendre jusqu'à 4 minutes, une fois la procédure de mise à jour terminée, le code affichera un message pour avertir l'utilisateur

Recherche dans le document

Le document permet à l'utilisateur de chercher des éléments qui répondent à des critères.

Naviguer entre les postes responsables

Les APRTs ont généralement sous leur responsabilité deux postes responsables. Le document est donc divisé en deux pages, chaque page a un tableau avec les informations d’un poste responsable. Pour naviguer entre les deux postes responsables, il suffit de cliquer en bas à gauche sur le poste qui intéresse l’utilisateur.

Rechercher les éléments en fonction du type d'ordre

L’utilisateur peut cliquer sur la liste déroulante et choisir le type d’ordre qu’il veut.

Rechercher les éléments en fonction de l'ordre

L’utilisateur peut marquer l’odre qui l’intéresse, pour n’afficher que celui-ci. Il peut aussi marquer un ordre partiellement complet pour afficher tous les ordres qui possèdent la combinaison de chiffres marquée dans la cellule de recherche.

Rechercher les éléments en fonction de la date de début

L'utilisateur peut choisir d'afficher les éléments avec une date de début plus grande ou plus petite que xxxx-xx-xx.Il est aussi possible pour l’utilisateur d’afficher uniquement les éléments qui sont compris dans une période pour les dates de début. Exemple : Les éléments avec une date de début plus grande que 2018-01-01 et plus petite que 2019-01-01.

Rechercher les éléments en fonction de l'Avis de maintenance

L’utilisateur peut marquer une combinaison de lettres et le script affichera tous les Avis de maintenance avec cette combinaison de lettres. La même chose s’applique pour la « Désignation », « Travail », « Note équipée ordonn », « Notes equi. Pre. Tra” & “Commentaire”.

Rechercher les éléments en fonction du Travail (Nombre d'heures de la tâche)

L’utilisateur peut marquer un chiffre X, pour n’afficher que les éléments avec ce nombre d’heures. L’utilisateur peut aussi choisir d’afficher les éléments plus grands ou plus petits que X.

Rechercher les éléments en fonction de plusieurs critères

L’utilisateur peut faire une combinaison des critères abordés plus haut, pour afficher des éléments spécifiques. Exemple : Afficher les éléments avec une date de début entre 2018-01-01 & 2019-01-01, avec moins de 15 heures de travail.

Ajout/Suppression d'un commentaire

Il est possible d’ajouter et de supprimer un commentaire. Les commentaires ne se trouvent pas dans SAP, ils sont seulement présents dans le fichier Excel. Ces derniers restent présents même avec une mise à jour.

Ajouter un commentaire

L’utilisateur peut ajouter un commentaire à côté de l’élément qui l’intéresse.

Supprimer un commentaire

L’utilisateur peut supprimer un commentaire de la même façon qu’il supprime du texte sur Word, il doit simplement cliquer sur la cellule qui l’intéresse

Modification du code pour modifier le fichier

Il est pertinent de mentionner que la maitrise de toutes les sections qui suivent n’est pas nécessaire pour l’utilisation de l’outil. Les sections plus bas sont un guide pour modifier le code VBA, de sorte à ajouter de l’information ou à en enlever.

Ajout d'une colonne au tableau

Il est possible de modifier le code pour ajouter une colonne supplémentaire venant de l’extraction SAP d’origine, pour pouvoir voir plus d’information dans les tableaux présents dans le document Excel.

Afficher la page «Informations_sauvegardees»

La page «Informations_sauvegardees» contient l’extraction SAP et toutes les colonnes de l’extraction des colonnes. Le code par la suite sélectionne les colonnes qui lui sont pertinentes pour les afficher dans les tableaux. Cette page est invisible, il est utile de la rendre visible pour savoir dans quelle colonne se trouve l’information à ajouter

Afficher le titre des colonnes et le quadrillage des pages «Informations_de_xxxxx»

La visualisation des colonnes est nécessaire pour savoir où se trouve la colonne que l’on veut ajouter l’information de la page

Informations sur la macro «copie_des_Informations_de_XXXXX» et «copie_des_Informations_de_YYYYY»

Les deux macros servent à copier les colonnes de la page «Informations_sauvegardees» et de la coller dans les deux pages avec les tableaux. Chaque macro se charge d’une page

Modifier la macro «copie_des_Informations_de_XXXXX» et «copie_des_Informations_de_YYYYY»

Avant, d’ajouter un élément, dans le cas de cet exemple, devant « Unité de charge », il faut décaler les colonnes concernées dans le code. Par la suite, on peut ajouter la nouvelle colonne.

Modifier la macro «enregistrement_Des_comentaire_XXXXX» et «enregistrement_Des_comentaire_YYYYY»

Ces deux scripts passent à travers la colonne avec les commentaires pour les sauvegarder, afin de pouvoir les réattribuer une fois la mise à jour terminée. La suppression ou l’ajout d’une colonne change la colonne où se trouvent les commentaires, de ce fait, il faut changer la position dans le code pour la nouvelle position où se trouvent les commentaires.

Modifier la macro «recherche()» et Faire les dernières touches

Ce script, «recherche()», permet de faire la recherche dans les tableaux en fonction des critères. L’ajout ou la suppression change la position du tableau, de ce fait l’utilisateur doit aller dans le code et donner les nouvelles positions du tableau.

Ajout d'un critère de recherche

Avant de pouvoir modifier le script, il faut activer certains éléments d’Excel.

Décaler des éléments dans la macro

Avant de prendre en considération, le nouvel élément ajouté, il faut adapter le code pour prendre en considération le décalage vers la droite. regarder certaines vidéos en ligne

Ajouter le nouveau critère dans le code (pour des lettres)

Si l’utilisateur veut chercher dans la colonne la présence de certaines lettres ou de certains mots, alors l’utilisateur doit suivre cette vidéo.

Ajouter le nouveau critère dans le code (pour des chiffres)

Si l’utilisateur veut chercher dans la colonne pour des valeurs numériques, par exemple toutes les valeurs plus grandes que 8 (>8), alors l’utilisateur doit suivre cette vidéo.

Redimensionner les dimensions du tableau de critère dans le code

Lorsque la recherche est lancée, le code regarde les éléments dans le tableau de critère. En raison que la longueur du tableau vient de changer à cause de l’ajout d’un nouveau critère, il faut changer la longueur du tableau de critère dans le code.

Mise en forme des tableaux

Cette section consiste à mettre à jour l’apparence des tableaux dans les deux pages. Il est important que la vidéo change l’apparence d’une seule page, il suffit de faire les mêmes étapes pour l’autre page.

Ajouter les nouvelles limites dans «ThisWorkbook»

Cette section se charge de mettre à jour les nouvelles limites du tableau de critère dans le code qui se charge de la mise à jour automatique.
